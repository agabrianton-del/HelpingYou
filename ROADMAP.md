# Roadmap

## Visión
HelpingYou es un proyecto en crecimiento. Este roadmap define prioridades sugeridas para orientar la evolución del repositorio de forma incremental y sostenible.

## Corto plazo
- Aclarar el propósito del proyecto en `README.md`.
- Documentar requisitos, instalación y forma de uso.
- Revisar la estructura del repositorio y estandarizar nombres de archivos y carpetas.
- Añadir una licencia si todavía no existe.
- Definir una primera estrategia de versionado.

## Mediano plazo
- Incorporar pruebas automatizadas para las funcionalidades principales.
- Configurar integración continua para validar cambios automáticamente.
- Mejorar manejo de errores, logs y observabilidad básica.
- Documentar arquitectura, decisiones técnicas y flujo de contribución.
- Identificar y reducir deuda técnica prioritaria.

## Largo plazo
- Establecer métricas de calidad y rendimiento.
- Preparar un flujo de releases y changelog.
- Evaluar automatizaciones adicionales para despliegue o mantenimiento.
- Fortalecer seguridad: dependencias, secretos y validaciones.
- Crear una hoja de ruta pública basada en feedback y uso real.

## Hitos sugeridos
### Hito 1: Base del proyecto
- README completo
- ROADMAP inicial
- Licencia
- Estructura clara del repositorio

### Hito 2: Calidad de desarrollo
- Linting/formateo
- Pruebas básicas
- CI inicial
- Guía de contribución

### Hito 3: Madurez operativa
- Releases
- Changelog
- Observabilidad
- Revisión de seguridad

## Notas
Este archivo puede evolucionar a medida que se aclaren los objetivos funcionales y técnicos del proyecto.
